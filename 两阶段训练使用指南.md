# 两阶段训练使用指南

## 📖 功能说明

两阶段训练策略结合了标准训练的快速收敛优势和鲁棒训练的边界优化能力：

- **阶段1（标准训练）**：快速学习数据特征，达到较高的分类准确率
- **阶段2（鲁棒性微调）**：使用 CROWN-IBP 损失函数优化模型的鲁棒性边界

### 优势
1. ✅ **训练效率高**：阶段1避免了一开始就计算昂贵的边界
2. ✅ **鲁棒性强**：阶段2专门针对鲁棒性进行优化
3. ✅ **灵活可调**：可以独立控制两个阶段的超参数

---

## 🚀 使用方法

### 基本用法（命令行）

```bash
cd robustness_verification

# 最简单的两阶段训练（使用默认参数）
python train.py --two-stage

# 自定义参数的两阶段训练
python train.py --two-stage \
    --dataset MNIST \
    --model simple \
    --epochs-stage1 5 \
    --epochs-stage2 5 \
    --lr 0.001 \
    --lr-stage2 0.0001 \
    --batch-size 128 \
    --bound-method CROWN-IBP \
    --epsilon 0.1 \
    --lambda-weight 1.0
```

### 参数说明

| 参数 | 说明 | 默认值 | 建议范围 |
|------|------|--------|---------|
| `--two-stage` | 启用两阶段训练模式 | - | - |
| `--dataset` | 数据集 (MNIST/CIFAR10) | MNIST | - |
| `--model` | 模型类型 (simple/medium/deep) | simple | - |
| `--epochs-stage1` | 第一阶段轮数 | 5 | 3-10 |
| `--epochs-stage2` | 第二阶段轮数 | 5 | 3-10 |
| `--lr` | 第一阶段学习率 | 0.001 | 0.0001-0.01 |
| `--lr-stage2` | 第二阶段学习率 | 0.0001 | 0.00001-0.001 |
| `--batch-size` | 批量大小 | 128 | 64-256 |
| `--bound-method` | 边界计算方法 | CROWN-IBP | IBP/CROWN/CROWN-IBP/alpha-CROWN |
| `--epsilon` | 扰动半径 | 0.1 | 0.01-0.3 |
| `--lambda-weight` | 鲁棒性损失权重 | 1.0 | 0.1-2.0 |

## 📈 与其他训练方式对比

| 训练方式 | 命令 | 优点 | 缺点 | 适用场景 |
|---------|------|------|------|---------|
| **标准训练** | `python train.py --dataset MNIST` | 速度快，准确率高 | 无鲁棒性保证 | 基线对比 |

## ⚠️ 注意事项

1. **训练时间**：
   - 阶段2比阶段1慢很多（因为要计算边界）
   - 如果时间紧张，可以减少 `epochs_stage2` 或使用更快的 `bound-method`（如 IBP）

2. **学习率选择**：
   - `lr_stage2` 应该比 `lr_stage1` 小 5-10 倍
   - 如果阶段2损失不下降，尝试更小的学习率

3. **内存占用**：
   - 阶段2需要更多内存（因为要创建 BoundedModule）
   - 如果内存不足，减小 `batch_size` 到 64 或 32

4. **数值稳定性**：
   - 如果训练中出现 NaN，减小 `lambda_weight` 或 `epsilon`
   - 检查 `lower_bounds` 是否出现异常值

## 🔧 故障排除

### 问题1：阶段2训练非常慢
**解决方案**：
```bash
# 使用更快的边界计算方法
python train.py --two-stage --bound-method IBP

# 或减少验证频率（需要修改代码）
```

### 问题2：阶段2准确率大幅下降
**解决方案**：
```bash
# 减小lambda_weight，降低鲁棒性损失的权重
python train.py --two-stage --lambda-weight 0.5

# 或增加阶段1的epoch数，让模型更好地收敛
python train.py --two-stage --epochs-stage1 10 --epochs-stage2 5
```

### 问题3：内存不足
**解决方案**：
```bash
# 减小batch size
python train.py --two-stage --batch-size 64
```

## 📝 代码实现要点

两阶段训练的核心实现在 `train_two_stage()` 函数中：

1. **阶段1**：复用 `train_model()` 的逻辑，标准交叉熵损失
2. **阶段2**：复用 `train_model_with_ab_crown()` 的逻辑，添加 VerificationLoss
3. **关键区别**：
   - 阶段2使用更小的学习率（`lr_stage2`）
   - 阶段2初始化 `VerificationLoss` 对象
   - 两个阶段共享同一个模型实例，权重连续更新

## 🎓 学术价值

两阶段训练的创新点：

1. **训练策略创新**：分离了"学习特征"和"优化边界"两个目标
2. **效率提升**：避免了从头开始计算昂贵边界的开销
3. **性能平衡**：在标准准确率和鲁棒性之间取得更好的权衡
4. **可扩展性**：可以轻松扩展到多阶段或其他鲁棒性方法
